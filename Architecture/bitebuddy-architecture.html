Key Data Flows
01
🍽️ Restaurant Discovery
  • Session created → participants' locations collected
  • Backend computes geographic centroid (midpoint)
  • Yelp Fusion queried with midpoint + session filters
  • Results filtered, ranked, shuffled → persisted to DB
  • All clients read the same ordered queue from DB
02
👆 Swipe & Match Detection
  • User swipes right → POST /swipes recorded in DB
  • Backend runs match query: COUNT(likes) = total_participants
  • On match → Realtime broadcast to session:{id} channel
  • All clients receive MATCH_FOUND event simultaneously
  • Session marked completed, result saved to history
03
🔐 Auth Flow
  • User signs in via Supabase Auth → JWT issued
  • POST /auth/sync called → users table upserted
  • JWT attached to all API requests as Bearer token
  • RLS policies enforce per-user data access in Postgres
04
👥 Session Invite Flow
  • Host creates session → 6-char invite code generated
  • Deep link shared: bitebuddy://session/XK92P1
  • Friends tap link → POST /sessions/:id/join
  • Lobby shows presence of all connected participants
  • Host starts session → restaurant fetch triggered
